# AI Based Voice Stress Analysis System for Lie Possibility Detection

This is a final-year project built with Python and Streamlit. The system analyzes English voice answers and estimates Lie Possibility using voice stress signs, pauses, hesitation, speaking speed, voice change, and answer matching.

The system does not claim that it can prove whether a person is lying. The final result is shown as:

- Low Lie Possibility
- Medium Lie Possibility
- High Lie Possibility

The result always includes this safety note:

> This result is only an estimate. It does not prove that the person is lying.

## Main Features

- English-only voice test flow
- Normal voice sample comparison
- Initial statement recording
- Adaptive follow-up questions based on the first statement
- Microphone recording through Streamlit when supported
- Audio upload fallback for WAV, MP3, and M4A files
- Speech-to-text using faster-whisper
- Manual transcript fallback if speech-to-text cannot create text
- Voice stress, pause, hesitation, speed, and voice change scoring
- Answer matching checks across the statement and follow-up answers
- Final Lie Possibility score and result level
- Downloadable PDF report
- Dark mode and light mode readable UI

## Installation Steps

Open Command Prompt or VS Code terminal inside the project folder.

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

The first speech-to-text run may download the `base.en` faster-whisper model. Keep internet available for the first run. After the model is available locally, the app can continue using it from the local cache.

## How the System Works

### 1. Normal Voice Sample

The user first records or uploads a normal voice sample. This gives the system a reference for the user's normal speaking pattern.

### 2. Initial Statement

The user speaks one statement about any real activity or situation. The system converts the voice into English text using faster-whisper when available.

Example:

```text
I stayed at home yesterday and studied Python.
```

The app then creates follow-up questions based on that statement.

### 3. Adaptive Follow-up Questions

The system checks the first statement for activity, place, time, and people clues. Based on those clues, it asks follow-up questions.

Example generated questions:

```text
What time did you start studying?
Which subject or topic were you studying?
Where were you studying?
What time did you stop studying?
```

If the first statement is about travel, work, shopping, or staying at home, the follow-up questions change accordingly.

### 4. Voice Feature Analysis

The audio feature module extracts:

- Duration
- RMS energy
- Zero crossing rate
- Spectral centroid
- MFCC mean values
- Silence ratio
- Estimated pause count
- Speaking stability score
- Pitch estimate

The UI shows readable labels instead of raw technical tables.

### 5. Answer Matching

The system checks whether the answers are clear and consistent enough. It looks for:

- Time review points
- Activity shifts
- Very short answers
- Repeated hesitation words
- Unclear answers
- Repeated words

The visible result is shown as:

```text
Matched
Not Fully Matched
```

### 6. Final Score

The final Lie Possibility score uses this structure:

```text
30% Voice Stress
25% Pause and Hesitation
25% Difference from Normal Voice
20% Answer Matching
```

Score categories:

```text
0% - 35%  = Low Lie Possibility
36% - 65% = Medium Lie Possibility
66% - 100% = High Lie Possibility
```

## Limitations

- The system estimates Lie Possibility only.
- It does not prove that a person is lying.
- Microphone quality, background noise, nervousness, and short answers can affect results.
- Speech-to-text quality depends on audio clarity and English pronunciation.
- The adaptive question planner uses rule-based logic, not an online AI API.
- The project is suitable for academic explanation and research-style voice behavior analysis.


faster-whisper converts English voice answers into text so the system can check answer matching without requiring the user to type every answer.
